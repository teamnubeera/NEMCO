EC2 Key Concepts and Terminologies

Instances

An Amazon EC2 instance is a virtual server in Amazon's Elastic Compute Cloud (EC2) for running applications on the AWS infrastructure. EC2 instances can be launched with various configurations for CPU, memory, storage, and networking.

  • Types of Instances: Standard, Compute Optimized, Memory Optimized, Storage Optimized, etc.
  • Instance States: Pending, Running, Stopped, Terminated
  • Launch Configurations: Choose OS, instance type, storage, and networking options.

Amazon Machine Images (AMIs)

An Amazon Machine Image (AMI) is a pre-configured template for launching EC2 instances. AMIs include the operating system, application server, applications, and related configurations.

  • AMI Types: Public AMIs, Custom AMIs, and Marketplace AMIs.
  • Region-Specific: AMIs are available in specific regions.
  • Creating Custom AMIs: Save the configuration of an existing instance to launch more instances with the same configuration.

Elastic Block Store (EBS)

Amazon Elastic Block Store (EBS) provides block-level storage volumes for use with EC2 instances. EBS volumes are used for storing data that requires frequent updates and low-latency access.

  • Persistent Storage: EBS volumes persist beyond instance termination.
  • Volume Types: SSD-backed, HDD-backed, and other specialized types (e.g., io1, gp2, st1).
  • Snapshots: EBS snapshots allow you to back up and restore data.

Elastic IP Addresses

Elastic IP Addresses (EIP) are static IPv4 addresses designed for dynamic cloud computing. An EIP is associated with your AWS account and can be associated with EC2 instances as needed.

  • Static IP: Unlike regular dynamic IP addresses, EIPs do not change when you stop and start your instance.
  • Association: You can associate an EIP to any EC2 instance within your account.
  • Reallocation: EIPs can be reassigned to other EC2 instances.

Security Groups

A security group acts as a virtual firewall for your EC2 instance to control inbound and outbound traffic. It defines which traffic is allowed to and from your instance.

  • Stateful: Inbound and outbound rules are evaluated separately.
  • Allow Rules: Security groups only allow traffic based on defined rules (no deny rules).
  • Multiple Security Groups: EC2 instances can be associated with multiple security groups for flexible access control.
