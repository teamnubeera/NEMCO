EC2 Instance Types Overview

General Purpose Instances

  • Example Families: t3, t3a, m5, m5a, m6g
  • Use Cases: Web servers, app servers, small to medium databases, development and testing environments.

Hands-on Activity:

Create a General Purpose EC2 instance using the m5 family in AWS Console. Test hosting a simple website using Apache.

Compute Optimized Instances

  • Example Families: c5, c5a, c6g
  • Use Cases: High-performance web servers, batch processing, gaming servers, and high-performance computing (HPC) workloads.

Hands-on Activity:

Launch a Compute Optimized EC2 instance using the c5 family and run a benchmarking tool like sysbench to test CPU performance.

Memory Optimized Instances

  • Example Families: r5, r5a, r6g, x1e
  • Use Cases: In-memory databases (e.g., Redis, Memcached), real-time analytics, big data processing.

Hands-on Activity:

Launch a Memory Optimized EC2 instance using the r5 family and deploy an in-memory database like Redis.

Storage Optimized Instances

  • Example Families: i3, i3en, d2
  • Use Cases: NoSQL databases, data warehousing, log processing, and data mining.

Hands-on Activity:

Launch a Storage Optimized EC2 instance using the i3 family and deploy a high-performance database such as Apache HBase.

GPU Instances

  • Example Families: p3, p4, g4dn
  • Use Cases: Machine learning (ML), deep learning, scientific computing, video rendering, and other GPU-intensive applications.

Hands-on Activity:

Launch a GPU EC2 instance using the p3 family and deploy a deep learning model using TensorFlow or PyTorch.
