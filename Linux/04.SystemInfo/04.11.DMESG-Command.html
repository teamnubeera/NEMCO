Understanding the Linux dmesg Command

The dmesg command in Linux is used to print or control the kernel ring buffer. It displays the messages generated by the kernel, which contain information about hardware devices, system startup, drivers, errors, and other kernel-related events.

Basic Syntax

dmesg [OPTION]...
  • OPTION: A specific option that alters the output format (e.g., -T for timestamps).

  • [ 0.000000]: The timestamp when the message was logged. This is shown in seconds since the system booted.
  • Booting Linux on physical CPU 0x0: A message showing the start of the Linux kernel on CPU 0.
  • Linux version: Information about the kernel version, build system, and compiler used to build it.
  • Command line: The kernel boot parameters passed during boot.
  • KERNEL supported cpus: Information on the supported CPUs by the kernel.

  • -c file (load messages from file): Load kernel ring buffer messages from a specified file instead of the default ring buffer. 
    dmesg -c
    • [    0.000000] Booting Linux on physical CPU 0x0

    dmesg -n 3

Additional Notes

  • dmesg is useful for viewing hardware-related logs, especially during the boot process or when troubleshooting kernel-level errors.
  • The dmesg command can also be helpful in diagnosing device driver issues, system crashes, or other kernel-related problems.
  • Messages displayed by dmesg include information about hardware initialization, network interfaces, and storage devices.
