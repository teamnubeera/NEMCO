Linux top Command

The top command in Linux is a powerful tool for monitoring system performance. It displays real-time information about running processes, CPU and memory usage, and other system statistics.

Basic Syntax:

top [OPTIONS]

Options:

  • -d <seconds>: Sets the delay between updates (default is 3 seconds).
  • -p <PID>: Monitors a specific process by its PID.
  • -n <iterations>: Limits the number of iterations before top exits.
  • -b: Runs top in batch mode for logging or scripting purposes.

Examples:

1. Display Real-Time Process Information:

$ top

Explanation: Displays an interactive interface with details about running processes, CPU, and memory usage.

2. Monitor a Specific Process:

$ top -p 1234

Explanation: Shows information about the process with PID 1234.

3. Batch Mode Output:

$ top -b -n 1

Explanation: Displays a snapshot of the current process information and exits after one iteration.

4. Set Custom Update Interval:

$ top -d 5

Explanation: Updates the display every 5 seconds instead of the default 3 seconds.

Output Breakdown:

  • Tasks: Total number of processes (running, sleeping, stopped, zombie).
  • CPU Usage: Percentage of CPU time spent in different states (user, system, idle).
  • Memory: Total, used, and free memory statistics.
  • Swap: Information about swap memory usage.
  • Process List: Details of running processes (PID, user, CPU%, memory%, command, etc.).

Practical Uses:

  • System Monitoring: Track CPU and memory usage in real-time.
  • Process Management: Identify resource-intensive processes.
  • Performance Tuning: Analyze system performance under different loads.

Script Example:

#!/bin/bash
# Save a snapshot of system performance
OUTPUT="system_snapshot.txt"
top -b -n 1 > $OUTPUT
echo "System performance snapshot saved to $OUTPUT"

Output:

System performance snapshot saved to system_snapshot.txt

Summary:

  • top: Provides real-time system and process monitoring.
  • -d: Sets update interval.
  • -p: Focuses on a specific process.
  • -n: Limits the number of iterations.
  • -b: Enables batch mode for logging or scripting.

Tip: Use htop, an alternative to top, for a more user-friendly interface.
