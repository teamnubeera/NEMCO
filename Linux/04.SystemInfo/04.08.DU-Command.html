Understanding the Linux du Command

The du (Disk Usage) command in Linux is used to estimate file space usage for directories and files. It summarizes the disk usage of each directory and its contents.

Basic Syntax

du [OPTION]... [FILE]...
  • If no file or directory is specified, du reports the disk usage of the current directory and its subdirectories.
  • If a specific file or directory is provided, du reports the disk usage for that file or directory.

  • 8: The disk usage of the directory or file (in kilobytes, by default).
  • ./dir1: The relative path of the directory or file.
  • 16: The total disk usage of the current directory (including subdirectories).

Commonly Used Options with du

  • -h (human-readable): Display sizes in a human-readable format, i.e., automatically using KB, MB, GB, etc., rather than just bytes. 
    du -h
    • 8.0K    ./dir1
4.0K    ./dir1/subdir1
4.0K    ./dir1/subdir2
16K     .
  • -s (summarize): Show only the total size of the specified directory or file (no subdirectories). 
    du -s
    • 16    .
  • -a (all files): Display the disk usage of all files, not just directories. 
    du -a
    • 8   ./dir1
4   ./dir1/subdir1
4   ./dir1/subdir2
4   ./dir1/file1
16  .
  • -c (total): Display a total at the end of the output. 
    du -c
    • 8   ./dir1
4   ./dir1/subdir1
4   ./dir1/subdir2
16  .
16  total
  • -d N (depth): Limit the depth of directory traversal to N levels. 
    du -d 2
    • 8   ./dir1
4   ./dir1/subdir1
4   ./dir1/subdir2
16  ./dir1
16  .

Example of du -h Output Explained

8.0K    ./dir1
4.0K    ./dir1/subdir1
4.0K    ./dir1/subdir2
16K     .
  • 8.0K: The disk usage of ./dir1 (8 KB).
  • 4.0K: The disk usage of ./dir1/subdir1 (4 KB).
  • 4.0K: The disk usage of ./dir1/subdir2 (4 KB).
  • 16K: The total disk usage of the current directory (including all subdirectories).

Additional Notes

  • du measures disk usage based on the number of disk blocks used by a file or directory. This may vary depending on the filesystem type.
  • The command reports disk usage for the file system where the directory resides, not the disk usage of any mounted file systems.
  • When you run du in a directory with a lot of files and subdirectories, it can take a while to complete.
