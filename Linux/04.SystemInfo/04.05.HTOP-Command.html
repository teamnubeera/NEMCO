Linux htop Command

The htop command in Linux is an interactive and user-friendly alternative to the top command. It provides a detailed and colorful interface to monitor system performance, including CPU, memory, and process usage.

Basic Syntax:

htop [OPTIONS]

Options:

  • -d <delay>: Sets the delay between updates in tenths of a second (e.g., -d 10 for 1 second).
  • -u <username>: Shows processes owned by a specific user.
  • -p <PIDs>: Monitors specific process IDs.
  • -s <column>: Sorts by a specified column (e.g., -s PERCENT_CPU).
  • -C: Starts htop in monochrome mode (disables colors).

Examples:

1. Launch htop:

$ htop

Explanation: Displays an interactive interface showing real-time process and system information.

2. Monitor Processes of a Specific User:

$ htop -u john

Explanation: Displays only the processes owned by the user john.

3. Sort Processes by Memory Usage:

$ htop -s PERCENT_MEM

Explanation: Sorts processes by memory usage in descending order.

4. Monitor Specific Processes:

$ htop -p 1234,5678

Explanation: Shows details of the processes with PIDs 1234 and 5678.

5. Disable Colors:

$ htop -C

Explanation: Runs htop in monochrome mode without colors.

Output Breakdown:

  • CPU Usage: Displays CPU usage for each core.
  • Memory and Swap: Shows real-time memory and swap usage.
  • Process Table: Lists processes with details such as PID, user, CPU%, memory%, and command.
  • Menu: Provides options for filtering, sorting, and process management.

Practical Uses:

  • System Monitoring: Keep track of system resources in real-time.
  • Process Management: Identify and terminate resource-heavy processes.
  • Customization: Use sorting and filtering to focus on specific details.

Keyboard Shortcuts:

  • F1: Help.
  • F2: Setup (customize display options).
  • F3: Search for a process by name.
  • F4: Filter processes.
  • F5: Tree view of processes.
  • F6: Sort processes by column.
  • F9: Kill a process.
  • F10: Exit htop.

Summary:

  • htop: Provides an interactive and user-friendly way to monitor system and process performance.
  • -u: Filters processes by user.
  • -s: Sorts processes by a specific column.
  • -p: Monitors specific PIDs.
  • -d: Sets the update delay.

Tip: Use htop for a more visually appealing and customizable interface compared to top.
