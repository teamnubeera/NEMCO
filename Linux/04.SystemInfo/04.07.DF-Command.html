Understanding the Linux df Command

The df (Disk Free) command in Linux is used to display the amount of disk space used and available on file systems. It reports the disk space usage for all mounted file systems, which includes the file system name, total size, used space, available space, and mount point.

Basic Syntax

df [OPTION]... [FILE]...
  • If no file or directory is specified, df reports the disk usage for all file systems.
  • If a specific file or directory is provided, df reports the disk space for the file system where that file or directory resides.

Common Output Columns

Here’s the typical output of the df command:

Filesystem     1K-blocks     Used Available Use% Mounted on
/dev/sda1       20474812  10234240  10240572  50% /
tmpfs            1024000        456   1023544   1% /dev/shm
/dev/sdb1       30720000   4096000  26624000  15% /mnt
  • Filesystem: The name of the file system (e.g., /dev/sda1, tmpfs, etc.). It represents the device or virtual file system.
  • 1K-blocks: The total size of the file system in kilobytes (KB), measured in 1,024-byte blocks.
  • Used: The amount of space used in the file system (in KB).
  • Available: The amount of free space still available for use (in KB).
  • Use%: The percentage of used space in the file system. This value helps you quickly understand how full the file system is.
  • Mounted on: The directory where the file system is mounted. This shows the location where the system can access the file system’s contents.

Commonly Used Options with df

  • -h (human-readable): Display sizes in a human-readable format, i.e., automatically using KB, MB, GB, etc., rather than just bytes. 
    df -h
    • Filesystem      Size  Used Avail Use% Mounted on
/dev/sda1        20G  10G  10G  50% /
tmpfs            1000M  500K  999M   1% /dev/shm
  • -T (file system type): Shows the file system type (e.g., ext4, xfs, etc.). 
    df -T
    • Filesystem     Type 1K-blocks     Used Available Use% Mounted on
/dev/sda1      ext4  20474812  10234240  10240572  50% /
tmpfs           tmpfs  1024000        456   1023544   1% /dev/shm
  • -i (inode information): Displays inode usage instead of block usage. This is useful for understanding how many files and directories are used versus available in the file system. 
    df -i
    • Filesystem      Inodes  IUsed   IFree IUse% Mounted on
/dev/sda1       1310720  500000  810000  38% /
tmpfs            256000     100  255900   1% /dev/shm

Example of df -h Output Explained

Filesystem      Size  Used Avail Use% Mounted on
/dev/sda1        50G   30G   15G  70% /
tmpfs            1.0G  500M  500M  50% /dev/shm
/dev/sdb1        100G  10G   85G  10% /mnt
  • /dev/sda1: A 50 GB partition, with 30 GB used, 15 GB free, and 70% used, mounted on the root (/) directory.
  • tmpfs: A temporary filesystem mounted on /dev/shm (often used for shared memory). It has 1 GB total space, with 500 MB used and 500 MB available.
  • /dev/sdb1: A 100 GB partition mounted on /mnt, with 10 GB used and 85 GB available, only 10% used.

Additional Notes

  • df works on file systems that are mounted and doesn't show information for file systems that are not currently mounted.
  • It doesn't show space usage on a specific directory unless you specify the directory, as in df /home.
