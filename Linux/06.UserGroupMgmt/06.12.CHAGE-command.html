chage: Change User Password Expiry Information

The chage command in Linux is used to modify the password expiry information for user accounts. System administrators typically use this command to configure password aging policies, such as setting maximum password age, minimum password age, and other related settings.

Basic chage Syntax

The basic syntax of the chage command is:

chage [options] [username]

Where:

  • [options] are optional flags that modify the behavior of the command.
  • [username] is the name of the user whose password expiry information you want to modify.

Examples

Display the Current Password Expiry Information for a User

To view the current password expiry settings for a user (e.g., john), use the following command:

chage -l john

Example output:

Last password change                                    : Jan 17, 2025
Password expires                                        : Apr 17, 2025
Password inactive                                       : May 17, 2025
Account expires                                         : Never
Minimum number of days between password change          : 7
Maximum number of days between password change          : 90
Number of days of warning before password expires       : 7

This output provides the following details about the user's password:

  • Last password change: The date of the last password change.
  • Password expires: The date when the password will expire.
  • Password inactive: The date when the account will become inactive (if applicable).
  • Account expires: The date when the account will be disabled (if set).
  • Minimum number of days between password change: The minimum time period that must pass before the user can change their password.
  • Maximum number of days between password change: The maximum time period allowed before the user is required to change their password.
  • Number of days of warning before password expires: The number of days before the password expires when the user will receive a warning to change it.

Change the Maximum Password Age

To change the maximum number of days a user can keep their password before it expires (e.g., set it to 60 days for user alice), use the following command:

sudo chage -M 60 alice

Here, the -M option specifies the maximum password age in days.

Set the Minimum Password Age

To set the minimum number of days before a user can change their password (e.g., set it to 5 days for bob), use:

sudo chage -m 5 bob

Here, the -m option specifies the minimum number of days between password changes.

Set the Password Expiry Warning Period

To set the number of days before the password expires that the user will receive a warning (e.g., set to 10 days for john), run:

sudo chage -W 10 john

Here, the -W option specifies the number of days of warning before password expiry.

Disable Password Expiry

If you want to disable password expiry for a user, you can use the -I option followed by 0 to set the password inactive period to never:

sudo chage -I 0 john

This will disable the password inactivity feature for the user john.

Set an Account Expiry Date

To set an account expiry date (e.g., set to Jan 1, 2026, for alice), use the -E option:

sudo chage -E 2026-01-01 alice

Here, the -E option specifies the date when the user account will expire and become inactive.

Useful chage Options

  • -l: List the current password expiry information for a user.
  • -M: Set the maximum number of days between password changes.
  • -m: Set the minimum number of days between password changes.
  • -W: Set the number of days of warning before the password expires.
  • -I: Set the number of days of inactivity before the account is disabled (set to 0 to disable the feature).
  • -E: Set the account expiry date (when the user’s account will be disabled).
  • -h: Display help information for the chage command.

Security Considerations

The chage command is a powerful tool for managing password and account expiry policies. Password aging can improve security by forcing users to change their passwords regularly. However, it’s important to ensure that password expiration policies are not too restrictive, as this may lead to users being locked out of their accounts.

Note: The chage command requires superuser privileges to modify password expiry settings for other users, but it can be used without elevated privileges to check information for the current user.

Conclusion

The chage command is an essential tool for Linux system administrators to manage password expiry information. By using it, you can enforce security policies that ensure users change their passwords at regular intervals, manage inactive accounts, and set account expiry dates.

For more advanced usage and additional options, refer to the official chage manual page.
