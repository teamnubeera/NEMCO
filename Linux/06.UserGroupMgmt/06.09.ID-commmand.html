id: Show User and Group Information

The id command in Linux displays information about a userâ€™s identity, including their user ID (UID), group ID (GID), and the groups to which they belong. This command is useful for understanding a user's access privileges and their group memberships on a system.

Basic id Syntax

The basic syntax of the id command is:

id [options] [username]

Where:

  • [options] are optional flags to modify the behavior of the command.
  • [username] is the name of a specific user whose information you want to view (optional).

Examples

Show Information for the Current User

To display information about the currently logged-in user, simply run:

id

Example output:

uid=1000(john) gid=1000(john) groups=1000(john),27(sudo)

This output shows:

  • uid=1000: The user ID (UID) of the user john.
  • gid=1000: The primary group ID (GID) of the user, which is usually the same as the UID.
  • groups=1000(john),27(sudo): The groups that the user belongs to. The user john belongs to the group john and the sudo group (which typically grants admin privileges).

Show Information for a Specific User

To display information about a specific user (e.g., alice), use the following command:

id alice

Example output:

uid=1001(alice) gid=1001(alice) groups=1001(alice),30(docker),50(staff)

Show Information with Additional Options

The id command can display more detailed information using various options.

Show Only the User ID (UID)

The -u option shows only the UID of the user:

id -u

Example output:

1000

Show Only the Group ID (GID)

The -g option shows only the GID of the user:

id -g

Example output:

1000

Show Only the Group Memberships

The -G option shows only the groups that the user belongs to:

id -G

Example output:

1000 27

Show All Information (Including Groups)

id command shows the UID, GID, and group memberships. However, you can also use -a to ensure it shows all details:

id -a

Useful id Options

  • -u: Show only the user ID (UID).
  • -g: Show only the group ID (GID) of the user's primary group.
  • -G: Show only the group IDs of all groups the user belongs to.
  • -n: Display the names of the user and group IDs instead of numeric values.
  • -a: Show all user and group information.
  • -h: Display help information for the id command.

Security Considerations

The id command provides detailed information about a user's identity and group memberships. Although the information is typically not sensitive, it can reveal group membership details that may be useful in auditing or security analysis. Always be mindful of who has access to user information on multi-user systems.

Note: The id command does not require elevated privileges to display information about the current user, but a user with sufficient privileges (e.g., root) can view information about any user on the system.

Conclusion

The id command is a simple yet powerful tool for displaying a user's identity, group memberships, and other related information on a Linux system. It's especially useful for administrators to quickly check group memberships and user IDs, which are important for managing permissions and security.

For more advanced usage and additional options, refer to the official id manual page.
