usermod: Modify a User Account

The usermod command is used in Linux to modify an existing user account. This allows system administrators to make changes to user details such as username, home directory, shell, and group memberships.

Basic usermod Syntax

The basic syntax of the usermod command is:

usermod [options] [username]

Where:

  • [options] are flags that specify the modifications you want to apply to the user account.
  • [username] is the name of the user whose account you want to modify.

Examples

Change the Username

To change the username of an existing user, use the -l option:

usermod -l new_username old_username

For example, to change the username of user john to johnny:

usermod -l johnny john

This will rename the user john to johnny, but it won't change the user's home directory name. You may want to rename the home directory as well using the -d option.

Change the User's Home Directory

To change the home directory of a user, use the -d option. You can also use the -m option to move the user's existing files to the new directory:

usermod -d /new/home/directory -m username

For example, to move the home directory of user alice to /home/alice_new:

usermod -d /home/alice_new -m alice

Change the User's Login Shell

If you want to change the login shell for the user, use the -s option followed by the path to the new shell:

usermod -s /bin/bash username

For example, to change the login shell for user bob to /bin/zsh:

usermod -s /bin/zsh bob

Add the User to a New Group

You can add the user to a new group using the -G option. This will add the user to the supplementary group(s) without affecting their primary group:

usermod -G new_group username

For example, to add the user charlie to the developers group:

usermod -G developers charlie

Change the User's Primary Group

To change the primary group of a user, use the -g option:

usermod -g new_primary_group username

For example, to change the primary group for user diana to admin:

usermod -g admin diana

Lock or Unlock the User Account

You can lock a user account to prevent login by using the -L option. To unlock the account, use the -U option:

usermod -L username   
usermod -U username

For example, to lock the account for user eve:

usermod -L eve

Useful usermod Options

  • -l: Change the username.
  • -d: Change the user's home directory.
  • -m: Move the user's files to the new home directory.
  • -s: Change the user's login shell.
  • -g: Change the user's primary group.
  • -G: Add the user to supplementary groups.
  • -a: Append the user to supplementary groups (use with -G).
  • -L: Lock the user account (disables login).
  • -U: Unlock the user account (enables login).
  • -e: Set the account expiration date.

Security Considerations

The usermod command is a powerful tool for modifying user accounts. However, be cautious when modifying users, especially when changing home directories, login shells, or group memberships. Ensure that the user has the appropriate permissions and access rights to perform their tasks.

Note: The usermod command typically requires elevated privileges (e.g., using sudo) to modify user accounts.

Conclusion

The usermod command is essential for system administrators when managing user accounts. It provides a flexible way to modify users' settings such as username, home directory, login shell, group memberships, and more. Understanding how to properly use the usermod command is crucial for maintaining a secure and well-managed system.

For more advanced usage and additional options, refer to the official usermod manual page.
