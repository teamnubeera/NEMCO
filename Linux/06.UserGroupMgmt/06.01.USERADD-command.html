useradd: Add a New User

The useradd command is used in Linux to create a new user account. This is essential for managing user access to the system, particularly when you want to add new users or manage system permissions.

Basic useradd Syntax

The basic syntax of the useradd command is:

useradd [options] [username]

Where:

  • [options] are optional flags to modify the behavior of the command (e.g., -m to create a home directory).
  • [username] is the name of the new user you want to create.

Examples

Create a New User

To create a new user without any additional options, use:

useradd username

For example, to create a user named john:

useradd john

This will create a new user account with the name john and set up the basic configuration. However, the user will not have a home directory or a password by default.

Create a New User with a Home Directory

If you want to create a new user and automatically create their home directory, use the -m option:

useradd -m username

For example, to create a user named alice with a home directory:

useradd -m alice

This will create the alice user and their home directory, usually located in /home/alice.

Create a New User and Set a Password

After creating a user, you can set their password using the passwd command:

passwd username

For example, to set the password for user alice:

passwd alice

You will be prompted to enter and confirm the new password for the user.

Create a New User with a Specific Shell

To specify a custom shell for the new user, use the -s option followed by the path to the shell:

useradd -s /bin/bash username

For example, to create a user bob with /bin/bash as their default shell:

useradd -s /bin/bash bob

Create a New User and Assign Them to a Group

You can specify which group the new user should belong to using the -g option:

useradd -g groupname username

For example, to create a user charlie and assign them to the developers group:

useradd -g developers charlie

Create a New User with a Specific UID

You can assign a specific user ID (UID) to the new user using the -u option:

useradd -u 1234 username

For example, to create a user diana with UID 1234:

useradd -u 1234 diana

Useful useradd Options

  • -m: Create the user's home directory if it does not exist.
  • -d: Specify the path to the home directory. If not specified, /home/username is used.
  • -s: Specify the shell that the user will use (e.g., /bin/bash).
  • -g: Assign the user to a specific group (use the group's name or GID).
  • -G: Add the user to additional groups.
  • -u: Set the user ID (UID) for the new user.
  • -e: Set the expiration date for the account (in YYYY-MM-DD format).
  • -p: Set an encrypted password for the user (though it's recommended to use passwd instead).

Security Considerations

The useradd command is typically used by system administrators to manage user accounts. It's important to ensure that you:

  • Assign appropriate permissions and access rights to users.
  • Use strong, unique passwords for each user account.
  • Be cautious when adding users to administrative groups like sudo or wheel to avoid unintentional privilege escalation.
Note: The useradd command may require elevated privileges (e.g., using sudo) to create new user accounts.

Conclusion

The useradd command is an essential tool for system administrators to add new users to a Linux system. Whether you're creating basic accounts or more complex configurations with custom shells, home directories, or user IDs, the useradd command provides a wide range of options for user account management.

For more advanced usage and additional options, refer to the official useradd manual page.
