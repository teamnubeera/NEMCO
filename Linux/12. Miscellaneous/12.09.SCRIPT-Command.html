script Command: Record a Terminal Session

The script command in Linux is used to record a terminal session. It records everything that is typed and displayed on the terminal, saving it to a file. This can be useful for documenting processes, creating logs, or reviewing terminal work later. The command creates a typescript file that can be replayed or reviewed for later reference.

Key Features of the script Command

  • Record terminal activity: The script command captures all output, including text and commands, to a file.
  • Create a log file: You can specify a file name to save the session, and all activities during that session are logged into this file.
  • Useful for documentation: It helps in documenting the exact terminal output for reports, tutorials, or troubleshooting.

Basic script Command Usage

The basic syntax for using the script command is:

script [file]

Where:

  • [file]: The name of the file where the session will be saved. If no file is specified, the default file name typescript will be used.

1. Record a Terminal Session

To start recording a session, simply run:

script

Everything that happens in the terminal (commands, outputs, errors, etc.) will be saved to a file named typescript by default.

2. Record and Specify the File Name

If you want to specify a custom file name to store the session, provide the name of the file:

script mysession.log

This will start recording the terminal session and save it to mysession.log in the current directory.

Common script Options

Here are some of the most frequently used options with the script command:

  • -a: Append to an existing file instead of overwriting it. For example: script -a mysession.log
  • -f: Flush the output after every write, ensuring that the file is updated in real-time. Example: script -f mysession.log
  • -q: Run the command in "quiet" mode, which suppresses the introductory message when starting the session. Example: script -q mysession.log
  • -t: Record the timestamp of each command executed. This is useful for tracking the time taken for each command. Example: script -t 2> timing.log mysession.log

Examples with Options

1. To record a session and save it to a file, appending to an existing file if needed:

script -a mysession.log

2. To run the script in quiet mode, suppressing the start message:

script -q mysession.log

3. To record the session with timestamps of each command:

script -t 2> timing.log mysession.log

Recording Output to Separate Files

By default, the script command records both the terminal output and any timing information into a single file. However, you can record the timing information separately. Use the -t option to store the timing data in one file and the session output in another, like this:

script -t 2> timing.log mysession.log

This will save the terminal output to mysession.log and the timing data to timing.log.

Stopping the Recording

To stop recording the session, simply type exit or press Ctrl + D. This will terminate the session and save the file.

Use Case Scenarios for script

  • Documentation: Record a terminal session to create documentation or tutorials. You can later play back the session to explain each step in the process.
  • Bug reporting: Capture the exact terminal output when encountering an issue so that it can be reviewed later or shared with others (e.g., developers or support teams).
  • Learning: Record your work while learning or experimenting with commands. This allows you to review your actions and outputs after the session ends.
  • Training: Use the script command to record a demo or training session for colleagues or trainees to review later.

Common Issues and Troubleshooting

  • File permissions: Make sure you have write permissions in the directory where you want to save the session file. If you get a "Permission Denied" error, use chmod or choose a different directory where you have write access.
  • Session not stopping: If you’re unable to stop the recording, check if there’s a process or command that’s still running in the background. Otherwise, Ctrl + D or exit should stop the session.
  • Long sessions causing large files: Long terminal sessions can generate large files. Consider using the -f option for real-time output flushing or breaking the session into smaller segments.
