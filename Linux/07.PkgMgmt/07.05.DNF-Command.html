YUM Command Tutorial

YUM (Yellowdog Updater, Modified) is a package management tool used in RPM-based Linux distributions like CentOS, Red Hat Enterprise Linux (RHEL), and Fedora. It is used to install, update, remove, and manage packages from software repositories, and it resolves dependencies automatically during package management operations.

Overview of YUM:

yum is an advanced front-end package management tool for RPM-based distributions. It works with package repositories and uses metadata to automatically resolve and install dependencies. YUM simplifies package management by automating updates, installations, and removals.

Basic Commands:

1. Installing Packages

To install a package, use the install command followed by the package name:

sudo yum install <package_name>

For example, to install vim:

sudo yum install vim

This command will automatically resolve dependencies and install the package.

2. Removing Packages

To remove a package from your system, use the remove command:

sudo yum remove <package_name>

For example, to remove vim:

sudo yum remove vim

This will uninstall the package and its dependencies if they are no longer required by any other installed packages.

3. Updating Packages

To update all installed packages to their latest available versions, use the update command:

sudo yum update

To update a specific package, specify the package name:

sudo yum update <package_name>

For example, to update vim:

sudo yum update vim

YUM will check the repositories for newer versions of the specified package and install them if available.

4. Searching for Packages

To search for a package, use the search command:

yum search <package_name>

For example, to search for a package related to vim:

yum search vim

This command will return a list of packages that match the search term along with brief descriptions.

5. Listing Installed Packages

To list all installed packages, use the list installed command:

yum list installed

This will show all packages that are currently installed on the system.

6. Displaying Package Information

To display detailed information about an installed package, use the info command:

yum info <package_name>

For example, to get information about the vim package:

yum info vim

This will display information such as the package description, version, size, and repository details.

7. Viewing Available Package Updates

If you want to check if there are any updates available for installed packages, use:

yum check-update

This command will list packages with available updates from the repositories without actually installing them.

8. Cleaning Up YUM Cache

Over time, YUM’s cache may accumulate a lot of unnecessary data. You can clean the cache using the clean command:

sudo yum clean all

This will remove all cached files, which can help free up disk space.

How YUM Works

yum works with repositories that contain RPM packages. When you run a command like yum install, it queries the repositories for the latest metadata about available packages and their dependencies. YUM then checks if the package is already installed and resolves any dependencies before installing the package.

YUM automatically installs dependencies, and if a package is removed, YUM can also remove unneeded dependencies (if no other installed package requires them). This makes YUM a convenient tool for managing system packages and updates.

Common Use Cases for YUM

1. Installing Software from Repositories

YUM is primarily used for installing packages from configured repositories. It’s useful when you need to quickly install software with automatic dependency resolution:

sudo yum install <package_name>

2. Updating the System

YUM is ideal for keeping the system up to date by running yum update. This ensures that you have the latest versions of software with bug fixes and security patches:

sudo yum update

3. Searching and Managing Packages

YUM makes it easy to search for packages and view detailed information. You can also list installed packages, check for updates, and even clean up your cache:

yum search <package_name>
yum list installed
yum check-update

4. Removing Unwanted Packages

If you want to uninstall packages that are no longer needed, YUM provides an easy way to remove them while resolving dependencies:

sudo yum remove <package_name>

Additional YUM Features

1. Managing YUM Repositories

You can manage your YUM repositories through configuration files located in /etc/yum.repos.d/. These files contain the repository URLs and other settings. To list all enabled repositories:

yum repolist

2. Installing from a Local RPM File

If you have a local RPM file, you can use YUM to install it (although rpm is the more direct method). The command is:

sudo yum localinstall <package_file.rpm>

This will automatically handle any dependencies that the RPM package requires.

3. Disabling a Repository Temporarily

If you need to disable a repository temporarily while running a YUM command, you can use the --disablerepo option:

yum --disablerepo=<repo_name> install <package_name>
