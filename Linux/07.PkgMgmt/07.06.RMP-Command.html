RPM Command Tutorial

RPM (Red Hat Package Manager) is a package management system used by many Linux distributions, especially those based on the Red Hat package format (e.g., Fedora, CentOS, RHEL, and openSUSE). RPM allows users to install, update, remove, and query individual packages, providing a powerful way to manage software packages on these systems.

Overview of RPM:

rpm is the low-level tool for managing RPM packages, and it works directly with the package files (.rpm). Unlike higher-level package managers like YUM or DNF, which handle dependencies and interact with repositories, RPM focuses on managing individual RPM packages on the system. It’s especially useful for installing, querying, verifying, and removing packages directly from RPM files.

Basic Commands:

1. Installing Packages

To install a package using RPM, you can use the -i (install) option, followed by the path to the RPM file:

sudo rpm -i <package_file.rpm>

For example, to install an RPM package named vim.rpm:

sudo rpm -i vim.rpm

This will install the package on your system, but RPM does not handle dependencies automatically. If the package has unmet dependencies, you’ll need to resolve them manually or use a higher-level package manager like YUM or DNF.

2. Removing Packages

To remove a package from your system, use the -e (erase) option followed by the package name:

sudo rpm -e <package_name>

For example, to remove the vim package:

sudo rpm -e vim

This will remove the package, but it will not automatically remove any dependencies that were installed with the package.

3. Querying Installed Packages

To query information about an installed package, use the -q (query) option:

rpm -q <package_name>

For example, to query information about the vim package:

rpm -q vim

This will return the version and architecture of the installed package. You can also use the -qi option for more detailed information:

rpm -qi vim

4. Listing Files in an RPM Package

If you want to list the files contained in an installed RPM package, use the -ql (query list) option:

rpm -ql <package_name>

For example, to list the files in the vim package:

rpm -ql vim

5. Verifying Installed Packages

To verify the integrity of an installed package, use the -V (verify) option:

rpm -V <package_name>

For example, to verify the integrity of the vim package:

rpm -V vim

This checks whether files from the package have been altered or are missing and reports any issues.

6. Checking Dependencies of an RPM Package

To check the dependencies of an RPM package, use the --requires option:

rpm -q --requires <package_name>

For example, to see the dependencies of the vim package:

rpm -q --requires vim

This will show you the libraries and other packages that the package depends on.

7. Checking What Files Belong to a Package

If you want to check which installed package owns a specific file, use the -qf (query file) option:

rpm -qf <file_path>

For example, to check which package provides the /usr/bin/vim file:

rpm -qf /usr/bin/vim

This will return the name of the package that installed the file.

8. Listing All Installed Packages

To list all installed packages on your system, use the -qa (query all) option:

rpm -qa

This will display a list of all packages currently installed on the system.

How RPM Works

rpm is a low-level package management tool that works directly with the RPM package files (.rpm). RPM does not manage dependencies or handle repositories, so it is typically used in combination with higher-level package managers like YUM or DNF, which automate the installation, update, and removal of packages while resolving dependencies automatically.

RPM provides several options for managing packages, including installing, querying, verifying, and removing packages. It can also be used to check for file ownership and dependencies, which is helpful for advanced system administration tasks.

Common Use Cases for RPM

1. Installing a Package from a Local RPM File

RPM is ideal for installing individual packages from local .rpm files. This is useful when downloading RPMs from third-party sources or when manually managing packages:

sudo rpm -i <package_file.rpm>

2. Querying Information About Installed Packages

Use RPM to check which version of a package is installed, or to list the files contained within a package:

rpm -qi <package_name>
rpm -ql <package_name>

3. Verifying Package Integrity

RPM’s -V option is helpful for system administrators who want to check if files from a package have been modified or tampered with:

rpm -V <package_name>

4. Removing an Installed Package

RPM can be used to remove a package from the system with the -e option. However, it does not automatically remove dependencies that were installed with the package:

sudo rpm -e <package_name>

Additional RPM Features

1. Querying Package Metadata

Besides querying installed packages, you can also query metadata in a package file using the -qpi option:

rpm -qpi <package_file.rpm>

This will show information about the package without installing it.

2. Installing RPM Packages with Dependencies

Although RPM does not handle dependencies on its own, you can use the --nodeps option to install a package without checking dependencies (use with caution):

sudo rpm -i --nodeps <package_file.rpm>

3. Upgrading an Installed Package

If you have a newer version of a package and wish to upgrade the installed version, you can use the -U (upgrade) option:

sudo rpm -U <package_file.rpm>

Summary of Key `rpm` Commands:

Command Description
sudo rpm -i <package_file.rpm> Install an RPM package from a local file.
sudo rpm -e <package_name> Remove an installed package from your system.
rpm -q <package_name> Query an installed package to check its version.
rpm -ql <package_name> List all files installed by a package.
rpm -qi <package_name> Show detailed information about an installed package.
rpm -V <package_name> Verify the integrity of an installed package.
rpm -qa List all installed packages.
