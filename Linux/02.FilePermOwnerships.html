Chapter 2: File Permissions and Ownership

This chapter covers essential commands in Linux for managing file permissions and ownership. Understanding these commands is crucial for controlling access to files and directories in a multi-user environment.

1. chmod - Change File Permissions

Purpose: Modifies the access permissions of a file or directory.

Usage:

  • chmod 755 file.txt - Sets the file permissions to read, write, and execute for the owner, and read and execute for others.
  • chmod +x file.sh - Adds execute permission to a file for the user (owner), group, and others.
  • chmod -r 644 file.txt - Removes write permissions for group and others while granting read/write permission to the owner.
  • chmod u+x file.txt - Adds execute permission for the file owner only.

Note: File permissions are represented by three sets of flags: r (read), w (write), and x (execute). These permissions can be modified using symbolic or numeric modes.

2. chown - Change File Owner and Group

Purpose: Changes the owner and/or group of a file or directory.

Usage:

  • chown user:file.txt - Changes the owner of "file.txt" to the user "user" while keeping the group unchanged.
  • chown user:group file.txt - Changes both the owner and group of the file.
  • chown -R user:group directory/ - Recursively changes the owner and group of all files and subdirectories within the specified directory.

3. chgrp - Change Group Ownership of a File

Purpose: Changes the group ownership of a file or directory.

Usage:

  • chgrp group file.txt - Changes the group ownership of "file.txt" to the specified group.
  • chgrp -R group directory/ - Recursively changes the group ownership of all files and subdirectories within the specified directory.

4. umask - Set Default File Permissions

Purpose: Sets default file permissions for newly created files and directories.

Usage:

  • umask 022 - Sets default permissions to rw-r--r-- for new files and rwxr-xr-x for directories.
  • umask 0777 - Sets no permissions for new files and directories (effectively preventing access).
