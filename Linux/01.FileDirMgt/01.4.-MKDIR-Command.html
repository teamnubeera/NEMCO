Linux mkdir Command

The mkdir command in Linux is used to create new directories. It allows you to create single or multiple directories at once, with various options for permissions and intermediate paths.

Basic Syntax:

mkdir [OPTIONS] DIRECTORY_NAME

Options:

  • mkdir: Creates a new directory.
  • mkdir -p: Creates parent directories as needed.
  • mkdir -m: Sets permissions for the new directory.
  • mkdir -v: Displays a message for each created directory.

Examples:

1. Basic Usage:

$ mkdir my_directory

Explanation: Creates a directory named my_directory in the current working directory.

2. Creating Parent Directories with -p Option:

$ mkdir -p /home/zaid/projects/new_project

Explanation: Creates the directory new_project and any intermediate directories (e.g., projects) if they do not exist.

3. Setting Permissions with -m Option:

$ mkdir -m 755 secure_directory

Explanation: Creates a directory named secure_directory with the specified permissions (755).

4. Verbose Mode with -v Option:

$ mkdir -v test_directory
mkdir: created directory 'test_directory'

Explanation: Displays a message confirming the creation of the directory.

5. Combining Options:

$ mkdir -p -m 700 /home/zaid/secure_folder/private

Explanation: Creates the directory private and its parent directories, setting the permissions to 700.

Practical Uses:

  • Organizing Files: Create directories to organize files and projects.
  • Scripting: Use in scripts to set up folder structures dynamically.
  • Setting Permissions: Create directories with specific permissions for secure data storage.

Script Example:

#!/bin/bash
# Create a directory structure for a project
mkdir -p /home/zaid/project/{docs,src,bin}
echo "Project directories created successfully."

Output:

Project directories created successfully.

Summary:

  • mkdir: Creates a new directory.
  • mkdir -p: Creates parent directories as needed.
  • mkdir -m: Sets permissions for the new directory.
  • mkdir -v: Displays messages for created directories.
