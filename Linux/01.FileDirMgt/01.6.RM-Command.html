Linux rm Command

The rm command in Linux is used to remove files or directories. It can delete both empty and non-empty directories, as well as individual files.

Basic Syntax:

rm [OPTIONS] FILE_OR_DIRECTORY

Options:

  • rm: Removes files.
  • rm -r: Removes directories and their contents recursively.
  • rm -f: Forces removal without confirmation prompts.
  • rm -i: Prompts for confirmation before each removal.
  • rm -v: Displays a message for each file or directory removed.

Examples:

1. Remove a File:

$ rm myfile.txt

Explanation: Deletes the file named myfile.txt.

2. Remove a Directory Recursively:

$ rm -r my_directory

Explanation: Deletes the directory my_directory and all its contents.

3. Force Remove Without Confirmation:

$ rm -f important_file.txt

Explanation: Deletes important_file.txt without asking for confirmation, even if it is write-protected.

4. Verbose Mode:

$ rm -v sample.txt
removed 'sample.txt'

Explanation: Displays a message confirming the removal of sample.txt.

5. Combining Options:

$ rm -rfv /home/zaid/temp_folder
removed '/home/zaid/temp_folder/file1.txt'
removed '/home/zaid/temp_folder'

Explanation: Removes the directory temp_folder and its contents forcefully, displaying verbose output.

Practical Uses:

  • Clearing Space: Remove unwanted files and directories to free up disk space.
  • Scripting: Automate file removal in scripts.
  • Cleaning Up: Delete temporary files and directories.

Script Example:

#!/bin/bash
# Remove log files older than 7 days
find /var/log -type f -name "*.log" -mtime +7 -exec rm -v {} \;
echo "Old log files removed."

Output:

removed '/var/log/system.log'
removed '/var/log/application.log'

Summary:

  • rm: Deletes files or directories.
  • rm -r: Removes directories and their contents recursively.
  • rm -f: Forces removal without confirmation.
  • rm -i: Prompts for confirmation before deletion.
  • rm -v: Displays messages for removed files or directories.

Warning: The rm command is powerful and should be used with caution to avoid unintentional data loss.
