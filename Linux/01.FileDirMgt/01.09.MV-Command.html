Linux mv Command

The mv command in Linux is used to move or rename files and directories. It can transfer files from one location to another or rename them within the same directory.

Basic Syntax:

mv [OPTIONS] SOURCE DESTINATION

Options:

  • mv: Moves or renames files and directories.
  • mv -i: Prompts before overwriting an existing file.
  • mv -f: Forces overwriting without confirmation.
  • mv -v: Displays verbose output for each operation.

Examples:

1. Move a File:

$ mv file1.txt /home/user/documents/

Explanation: Moves file1.txt to the /home/user/documents/ directory.

2. Rename a File:

$ mv oldname.txt newname.txt

Explanation: Renames oldname.txt to newname.txt within the same directory.

3. Move a Directory:

$ mv dir1 /home/user/projects/

Explanation: Moves the directory dir1 to /home/user/projects/.

4. Prompt Before Overwriting:

$ mv -i file.txt /backup/

Explanation: Asks for confirmation before overwriting any existing file in the /backup/ directory.

5. Verbose Mode:

$ mv -v data.txt archive/
'moved data.txt to archive/data.txt'

Explanation: Displays a message confirming the move operation.

6. Force Overwriting:

$ mv -f temp.txt /tmp/

Explanation: Overwrites temp.txt in the /tmp/ directory without confirmation.

Practical Uses:

  • File Management: Organize and relocate files and directories.
  • Renaming: Change file or directory names easily.
  • Scripting: Automate file movement and renaming in shell scripts.

Script Example:

#!/bin/bash
# Move all .log files to the logs directory
mkdir -p logs
mv -v *.log logs/
echo "Log files moved to the logs directory."

Output:

moved 'error.log' to 'logs/error.log'
moved 'access.log' to 'logs/access.log'

Summary:

  • mv: Moves or renames files and directories.
  • mv -i: Prompts before overwriting files.
  • mv -f: Forces overwriting without confirmation.
  • mv -v: Provides verbose output for each operation.

Warning: Be cautious when using the mv command to avoid unintentional overwriting or data loss.
