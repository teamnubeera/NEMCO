Linux cp Command

The cp command in Linux is used to copy files and directories. It can duplicate files from one location to another or create backups of files and directories.

Basic Syntax:

cp [OPTIONS] SOURCE DESTINATION

Options:

  • cp: Copies a file or directory.
  • cp -r: Copies directories and their contents recursively.
  • cp -i: Prompts before overwriting an existing file.
  • cp -f: Forces overwriting without confirmation.
  • cp -v: Displays verbose output for each file copied.
  • cp -u: Copies files only if the source is newer than the destination or if the destination file does not exist.

Examples:

1. Copy a File:

$ cp file1.txt file2.txt

Explanation: Copies file1.txt to file2.txt.

2. Copy a Directory Recursively:

$ cp -r source_dir target_dir

Explanation: Copies the directory source_dir and all its contents to target_dir.

3. Prompt Before Overwriting:

$ cp -i oldfile.txt newfile.txt

Explanation: Asks for confirmation before overwriting newfile.txt.

4. Verbose Mode:

$ cp -v data.txt backup.txt
'copied data.txt to backup.txt'

Explanation: Displays a message confirming the copy operation.

5. Force Overwriting:

$ cp -f important.txt archive/

Explanation: Overwrites important.txt in the archive directory without confirmation.

6. Copy Only Newer Files:

$ cp -u document.txt /backup/

Explanation: Copies document.txt to /backup/ only if it is newer or doesn't exist there.

Practical Uses:

  • File Backup: Create backups of important files and directories.
  • File Management: Organize and copy files to specific locations.
  • Scripting: Automate file copying in shell scripts.

Script Example:

#!/bin/bash
# Backup all .txt files from documents to backup folder
mkdir -p backup
cp -v documents/*.txt backup/
echo "Backup completed."

Output:

copied 'documents/file1.txt' to 'backup/file1.txt'
copied 'documents/file2.txt' to 'backup/file2.txt'

Summary:

  • cp: Copies files or directories.
  • cp -r: Copies directories recursively.
  • cp -i: Prompts before overwriting files.
  • cp -f: Forces overwriting without confirmation.
  • cp -v: Provides verbose output for each operation.
  • cp -u: Copies only newer files.

Warning: Be cautious when using the cp command to avoid unintentional overwriting of files.
