nice Command: Set the Priority of a Process

The nice command in Linux is used to set the priority of a process. In Unix-like systems, processes have a priority level known as "niceness," which determines the amount of CPU time they receive. By adjusting the niceness of a process, you can make it run with a higher or lower priority relative to other processes.

Key Features of the nice Command

  • Adjust process priority: The primary function of the nice command is to adjust the priority (niceness) of a process, either to increase its priority (make it "nicer") or decrease it (make it "less nice").
  • Priority range: Niceness values range from -20 (highest priority) to +19 (lowest priority), with 0 being the default priority.
  • Modify CPU usage: By adjusting the priority, you control how much CPU time a process gets compared to others. Higher priority processes are allocated more CPU time, while lower priority ones get less.

The nice command is typically followed by the command you want to execute with the adjusted priority. If no priority is specified, it will use the default niceness of 10.

1. Set a specific niceness value for a process

nice -n <niceness_value> <command>

The -n option allows you to specify the niceness value for the process. The value can be between -20 and +19, where lower values (closer to -20) mean higher priority, and higher values (closer to +19) mean lower priority.

Example: To run a command with higher priority:

nice -n -10 command_name

This will run command_name with a priority of -10 (higher priority than the default).

2. Use nice to run a command with default priority

nice command_name

If you don't specify a priority, nice will run the command with the default priority of 10.

Example: Running a command with the default priority of 10:

nice command_name

3. Run a command with the highest priority

nice -n -20 command_name

The -20 value represents the highest possible priority. This command will run the specified command with the highest priority, meaning it will receive the maximum available CPU time.

Example: Running a command with the highest priority:

nice -n -20 command_name

How niceness Affects Process Priority

In Linux, the niceness value is used to adjust the priority of a process. By default, processes are assigned a niceness of 0. The lower the niceness value (toward -20), the higher the priority, and the more CPU time the process will get. Conversely, the higher the niceness value (toward +19), the lower the priority, meaning the process will get less CPU time.

  • Niceness -20: Highest priority (maximum CPU time).
  • Niceness 0: Default priority (standard CPU time allocation).
  • Niceness +19: Lowest priority (minimal CPU time).

Processes with higher priority (lower niceness values) are given more CPU time by the operating system, while processes with lower priority (higher niceness values) may be delayed or given less CPU time, depending on the system load.

Examples of Using nice

  1. Run a command with default priority (niceness 10): nice command_name
  2. Run a command with higher priority (niceness -10): nice -n -10 command_name
  3. Run a command with the highest priority (niceness -20): nice -n -20 command_name
  4. Run a command with lower priority (niceness +10): nice -n +10 command_name

Commonly Used Commands in Conjunction with nice

  • renice: The renice command is used to change the priority (niceness) of an already running process. It allows you to adjust the priority of an active process rather than setting it when the process is started.
  • ps: The ps command can be used to view information about running processes, including their PID and priority (niceness). For example, ps aux lists all processes with their niceness values.
  • top: The top command displays an interactive list of running processes, including their priority and CPU usage. This is useful for monitoring the resource usage of processes in real-time.

Troubleshooting

  • Permission Issues: The nice command allows users to run commands with a lower niceness value (lower priority), but only privileged users (e.g., root) can set a niceness value of less than 0. Regular users can only increase the niceness (make the process lower priority).
  • High Priority Processes Not Responding: If you set a process to run with a very high priority, it might consume too much CPU time, which could result in other processes being delayed. Make sure to monitor the system load and adjust priorities as needed.
  • Process Not Starting with Desired Priority: Double-check that you are using the correct syntax and that the priority level falls within the allowed range of -20 to +19.

It is commonly used in high-performance computing or when running resource-intensive tasks that need to be balanced with other processes.
