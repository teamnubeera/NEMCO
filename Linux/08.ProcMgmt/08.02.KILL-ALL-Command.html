killall Command: Terminate Processes by Name

The killall command in Linux is used to terminate processes by their name, rather than their Process ID (PID). This is useful when you want to stop all processes that are running with a specific name, such as terminating all instances of a particular application.

Key Features of the killall Command

  • Terminate processes by name: Unlike the kill command, which requires the PID, killall allows you to terminate all processes by specifying their name.
  • Send signals to processes: You can send different signals to processes (like terminate, kill, stop) using the killall command.
  • Supports case-sensitive matching: By default, killall performs a case-sensitive search for the process name.

Basic killall Command Usage

Here are some common killall commands for managing processes by their name:

1. Terminate processes by name

killall <process_name>

This sends the default SIGTERM signal (signal number 15) to all processes matching the given name, requesting them to terminate.

Example: killall firefox will terminate all processes with the name "firefox".

2. Forcefully terminate processes by name

killall -9 <process_name>

This sends the SIGKILL signal (signal number 9) to forcefully terminate all processes with the specified name. The SIGKILL signal immediately terminates processes without allowing them to clean up resources.

Example: killall -9 firefox will forcefully terminate all processes with the name "firefox".

3. Send a different signal to processes

killall -<signal> <process_name>

You can specify a signal to send to the processes using the - option. Common signals include:

  • SIGSTOP (signal number 19): Suspend the processes.
  • SIGCONT (signal number 18): Resume suspended processes.
  • SIGINT (signal number 2): Interrupt the processes (like pressing Ctrl+C in the terminal).

Example: killall -SIGSTOP firefox will suspend all processes with the name "firefox".

4. Terminate processes by name with case-insensitive matching

killall -i <process_name>

By default, killall is case-sensitive when matching the process name. The -i option allows for case-insensitive matching of the process name.

Example: killall -i firefox will terminate processes named "Firefox", "firefox", or "FIREFOX".

5. List processes that match the name (without terminating)

killall -l <process_name>

This option lists all processes matching the name but does not terminate them. This can be useful for previewing which processes would be affected.

Example: killall -l firefox will list all processes matching the name "firefox" without terminating them.

Common Signals

The killall command can send various signals to processes, here are some common ones:

  • SIGTERM (15): Gracefully terminate the processes. This is the default signal sent by killall.
  • SIGKILL (9): Forcefully terminate the processes. This signal cannot be ignored by the process.
  • SIGSTOP (19): Suspend the processes (pause execution).
  • SIGCONT (18): Resume the suspended processes.
  • SIGINT (2): Interrupt the processes (equivalent to pressing Ctrl+C).
  • SIGHUP (1): Hang up. Often used to tell a daemon to reload its configuration.

Examples

  1. Gracefully terminate all processes by name: killall firefox (default SIGTERM signal).
  2. Forcefully terminate all processes by name: killall -9 firefox (SIGKILL signal).
  3. Suspend all processes by name: killall -SIGSTOP firefox (SIGSTOP signal).
  4. Resume all suspended processes by name: killall -SIGCONT firefox (SIGCONT signal).
  5. Terminate all processes with case-insensitive matching: killall -i firefox (case-insensitive match).

Troubleshooting

  • Permissions: Some processes may require root (administrator) permissions to terminate. In such cases, use sudo killall <process_name>.
  • Process not terminating: If a process does not terminate with killall, it may be ignoring the signal. Try using killall -9 <process_name> to forcefully terminate it.
  • Check if a process is still running: After issuing a killall command, check if the process is still running using ps aux, top, or pgrep.

Conclusion

The killall command is a useful tool for terminating processes by their name, rather than by PID. It allows you to send various signals to processes, including graceful termination and forceful termination. Understanding how to use killall effectively can help you manage system processes and terminate unresponsive programs by name.
