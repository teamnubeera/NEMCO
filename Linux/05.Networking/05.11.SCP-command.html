SCP: Secure Copy Files Between Hosts Over SSH

The scp (secure copy) command is a method to securely transfer files between two computers over an SSH connection. It is commonly used in Unix-like systems such as Linux, macOS, and other Unix-based OSes.

Basic SCP Syntax

The basic syntax for the scp command is:

scp [options] [source] [destination]

Where:

  • [options] are optional flags (e.g., -r for recursive copying of directories).
  • [source] is the path to the file or directory you want to copy. It can be on your local system or remote host.
  • [destination] is the path where you want to copy the file or directory to, which can also be local or remote.

Examples

Copy a File from Local to Remote

To copy a file from your local machine to a remote machine, you would use:

scp /path/to/local/file username@remote_host:/path/to/remote/directory

For example:

scp ~/Documents/myfile.txt user@192.168.1.10:/home/user/Documents/

Copy a File from Remote to Local

To copy a file from a remote server to your local system, you would reverse the source and destination:

scp username@remote_host:/path/to/remote/file /path/to/local/directory

For example:

scp user@192.168.1.10:/home/user/Documents/myfile.txt ~/Downloads/

Copy a Directory Recursively

If you want to copy an entire directory (including subdirectories and files), use the -r flag to copy recursively:

scp -r /path/to/local/directory username@remote_host:/path/to/remote/directory

Using a Specific Port

If the SSH service is running on a non-default port, use the -P option to specify the port number:

scp -P 2222 /path/to/local/file username@remote_host:/path/to/remote/directory

  • -r: Copy directories recursively.
